# Documentation

Start here. The docs are split into three kinds:

1. **The guide** (this directory) — living documents describing what the project *is* and
   *does today*: the data, the pipeline, the scoring methodology. Kept up to date as the
   codebase changes.
2. **[`results/`](results/)** — dated findings logs, one per analysis milestone. Immutable
   snapshots: they describe what the numbers said on that date, with the caveats that
   applied then.
3. **[`research/`](research/)** — the original research phase (2026-07-03/04), preserved
   verbatim. Contains every source, tool and architecture that was *considered*, including
   avenues not (or not yet) pursued — BBC scraping, OpenWeatherMap, Evidence/Observable
   dashboards, the `scores`/MAPIE libraries, gridded ECMWF data, and so on. Read it when
   deciding what to pick up next, not to learn how the current system works.

## Reading order (new to the project?)

| # | Doc | What it answers |
|---|---|---|
| 1 | [overview.md](overview.md) | What is this project? How do the pieces fit together? What has it found so far? |
| 2 | [data-sources.md](data-sources.md) | Where does the data come from? What is ground truth and why? Which forecast models? How does coverage differ by station/region? |
| 3 | [data-layout.md](data-layout.md) | What exactly is cached on disk, in what format, and what does every field mean? |
| 4 | [methodology.md](methodology.md) | How are forecasts scored? What do the metrics mean and what are the known caveats? |
| 5 | [glossary.md](glossary.md) | Every acronym and term of art (ERA5, UKMO, MOGREPS, ETS, conformal…). Keep it open in a tab while reading the others. |
| 6 | [results/](results/) | What the numbers actually said, in date order. |
